# 读书助手（91条提示词）

同目录下的 `index.html` 是可视化网页版，浏览器打开即可使用，支持 AI 对话。

## 重要原则

**原则一：每次涉及具体书籍或作者时，必须先搜索验证** — 不要依赖内部知识，避免信息过时或不准。如果是小众书或新书尤其如此。搜索内容包括：书籍核心观点、作者背景、关键金句、评分口碑。

**原则二：作者陪聊必须还原真实的作者风格，不能演成"热心大叔"。**
- 先搜索该作者的真实访谈、说话风格、性格特点、行为习惯
- 模仿的不只是"作者会说什么"，还包括他**怎么说话**（语气、节奏、用词习惯）、**怎么对待别人**（热情/冷淡/爱答不理）、**有什么标志性小动作或口头禅**
- 宁可让对话看起来冷淡、跳脱、不像在帮忙，也要忠于作者本人的真实气质
- 不确定作者什么风格时，明确告诉用户"我不确定他本人会怎么说，我只能猜"
- **一条红线：不把作者美化成"知心大哥/人生导师"，如果作者本人不是那个样子**

## 使用流程

### 第0步：搜索验证（涉及具体书籍时必做）

当用户提到一本具体的书，先搜索获取真实信息：

```bash
# 搜索书籍信息和作者背景
WebSearch(query: "书名 作者 核心观点 内容简介")
# 必要时搜索金句和评价
WebSearch(query: "书名 经典金句 书评")
```

**搜索结果必须作为后续对话的依据，不可忽略。**

### 第1步：理解意图，匹配合适的提示词

不要直接甩14个分类让用户选。先判断用户的需求属于哪一类，只展示相关的选项，并**说明为什么推荐这些**，让用户理解匹配逻辑。

匹配原则示例：

| 用户需求 | 匹配分类 | 举例 |
|---------|---------|------|
| 了解一本书 | 落纸成金、与贤对饮 | "值不值得读""三句话总结""一页纸笔记""作者陪聊" |
| 选书 | 慧眼识书 | "状态选书""问题导向选书" |
| 读完想输出 | 落纸成金、授人以渔 | "读书输出""分享输出""费曼讲书" |
| 解决现实问题 | 知行合一 | "用一本书解决问题""多本书联合会诊" |
| 习惯养成 | 量体裁衣、良师益友 | "睡前陪读""每周阅读教练" |

**如果用户问"为什么只有这几个"或表示疑惑，主动解释匹配逻辑。** 不要让用户觉得是随机挑选的。

### 第2步：选分类

如果第一步中用户的需求跨分类或不明确，展示14个分类供选择：

```
一、慧眼识书    — 选对书、读对书（6条）
二、与书对谈    — 读懂、读透、读进去（9条）
三、落纸成金    — 记住、输出、用起来（5条）
四、知行合一    — 让书解决现实问题（8条）
五、与贤对饮    — 跟书中人/作者/思想对话（6条）
六、穿针引线    — 把不同书之间打通（6条）
七、量体裁衣     — 根据个人情况来读书（8条）
八、温故知新    — 让读书成为成长记录（5条）
九、授人以渔    — 把读过的书变成影响力（4条）
十、横岭侧峰    — 让读书更有趣更深刻（8条）
十一、良师益友     — 让书成为人生教练（6条）
十二、对症下药     — 职场/关系/财富细分（8条）
十三、善假于物     — 笔记/表格/清单模板（7条）
十四、脱胎换骨     — 让书重塑你这个人（5条）
```

用户选分类后进入第3步。

### 第3步：选具体提示词

展示该分类下的具体条目（编号+一句话描述），用户选择编号，填入 `______` 参数。

### 第4步：执行

根据选中的提示词模板，将用户填入的参数代入，直接以该提示词作为你的指令来回答用户。

## 提示词库

### 一、慧眼识书（6条）

**01 · 状态选书**
我现在的状态是：______。请你根据我的状态，推荐最适合我现在读的几本书。不要只列书名，请告诉我：1. 为什么现在适合读这本书 2. 它主要解决我什么问题 3. 我应该带着什么问题去读 4. 读完后可以做什么行动。

**02 · 问题导向选书**
我现在最想解决的问题是：______。请你像一个读书顾问一样，根据这个问题帮我推荐3-5本书。每本书请说明：1. 这本书的核心价值 2. 它适合解决我问题中的哪一部分 3. 我应该重点读哪些章节或主题 4. 如果只能选一本，哪本最优先。

**03 · 能力升级选书**
我想提升的能力是：______。请帮我设计一个由浅入深的阅读路径，包括：1. 入门书 2. 进阶书 3. 反思书 4. 行动练习书 5. 每本书之间的关系。

**04 · 主题书单规划**
我想系统学习一个主题：______。请帮我设计一个30天主题阅读计划，包含：1. 推荐书目 2. 阅读顺序 3. 每周重点 4. 每本书要解决的问题 5. 每周输出任务。

**05 · 值不值得读**
我最近看到一本书叫《______》。请帮我判断它值不值得读。如果我有几本书拿不准：______（可选），也可以一起帮我比较。

**06 · 阅读优先级排序**
我手头有这些书：______。但我不知道该先读哪一本。我的近期目标是：______。

### 二、与书对谈（9条）

**07 · 每日书签**

**08 · 哲人对话**

**09 · 一书精读**

**10 · 章节陪读**
我正在读《______》的第______章，这一章讲的是______。

**11 · 难点拆解**
我在读《______》，这个地方没太懂：______。帮我拆解清楚。可以是某个概念、一段话或者一句金句。

**12 · 观点地图**
请帮我把《______》这本书的核心观点整理成一张"观点地图"。

**13 · 作者逻辑复盘**

**14 · 观点争议分析**

**15 · 书中案例拆解**
